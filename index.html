1. Declaring Variables

In JavaScript, you can declare variables using three keywords:

A. var (Oldest way):

• Function-scoped.
• Can be re-declared and updated within the same scope.
• Does not support block-scoping (it ignores {} boundaries).

B. let (Modern and recommended for mutable variables):

• Block-scoped.
• Can be updated but not re-declared in the same block.

C. const (Modern and for constants):

• Block-scoped.
• Must be initialized at the time of declaration.
• Cannot be updated or re-declared.

2. Variable Naming Rules

• Names can include letters, digits, underscores (_), and dollar signs ($).
• Must begin with a letter, $, or _.
• Are case-sensitive (myVar and myvar are different).
• Cannot use reserved keywords (e.g., var, let, class, etc.).

3. Variable Scope

Scope defines where a variable is accessible:

1.Global Scope: Variables declared outside any function or block are globally scoped.
2.Function Scope: Variables declared with var inside a function are only accessible within that function.
3.Block Scope: Variables declared with let or const inside {} are only accessible within that block.

4. Variable Hoisting

• Variables declared with var are “hoisted” to the top of their scope but remain uninitialized until the code execution reaches their declaration.
• Variables declared with let and const are also hoisted but cannot be used before their declaration due to the temporal dead zone.

5. Data Types in Variables

JavaScript variables can store different types of data:

1.Primitive Types:

• String: "Hello"
• Number: 42
• Boolean: true or false
• Undefined: A declared variable without a value.
• Null: Represents the absence of a value.
• Symbol: Unique and immutable value.
• BigInt: For very large integers.

2.Non-Primitive Types:

• Object: { key: "value" }
• Array: [1, 2, 3]
• Function: function() {}

6. Best Practices

• Use const by default. Only use let if you need to reassign the variable.
• Avoid var unless maintaining older code.
• Use descriptive names for variables.
